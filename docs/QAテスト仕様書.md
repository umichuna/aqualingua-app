## 1. はじめに

### 1.1 アプリ概要
- **アプリ名**: AquaLingua（水族館 × 英単語学習アプリ）
- **主な機能**: 単語帳の管理、3種類の学習モード（自己採点・4択クイズ・聞き流し）、水槽で魚を育てる、ショップ・ガチャ、図鑑、クラウド同期
- **技術構成（参考）**: Next.js（ウェブアプリの土台）／ PWA（スマホのホーム画面に追加して使えるアプリ形式）／ IndexedDB（端末の中にデータを保存するしくみ）＋ Azure SQL（クラウド上のデータ保管庫）を **LWW（Last Write Wins ＝ 後から保存した方が勝つ）** で同期

### 1.2 対象環境（テスト端末）
- **メイン**: スマホ（iPhone の Safari / Android の Chrome）
- **サブ**: PCブラウザ（Chrome / Edge）
- **同期テストは必ず2端末で実施**（例：スマホ＋PC、またはスマホ2台、もしくはPCの通常ウィンドウ＋シークレットウィンドウ）。両方とも **同じGoogleアカウント** でログインすること。

### 1.3 テストの進め方
1. 上の表から順に1項目ずつ操作する。
2. 各行の結果を **✅（合格）／❌（不合格）／保留** で記録する。
3. ❌になった項目は「第9章 コード調査レポート」と照らし合わせ、原因の見当をつける。
4. 「テスト種別」が **異常系** の行は、わざと間違った操作・極端な値を入れて、アプリが壊れず正しくエラー表示するかを見る。

### 1.4 事前準備
- テスト用データを用意：単語を5〜10語、魚を3〜4匹ほど。
- 2端末で同じGoogleアカウントにログインしておく。
- **PWAのキャッシュ更新手順**（古い画面が出るとき）：ホーム画面のアイコンを削除 → ブラウザでアプリURLを開く → ページ全体を再読み込み → 再度ホーム画面に追加。

### 1.5 用語メモ（カッコ内はやさしい言い換え）
- **pull（プル）**= クラウドから端末へデータを取ってくること
- **push（プッシュ）**= 端末からクラウドへデータを送ること
- **syncNow（手動同期）**= 設定画面の「☁️ 同期」ボタン。pull→push をまとめて実行する
- **ボックス**= 水槽に入りきらない魚を一時的に預けておく場所
- **通帳（ゴールド台帳）**= ゴールドの増減の記録

## 11. コード調査レポート（疑わしい箇所・修正の下準備）

> テストで ❌ が出たとき、原因の見当をつけるための一覧です。**優先度・該当コード・原因・修正方針**を整理しています（今回は実装修正は行いません）。

### 🔴 高（同期・データ整合性に直結）

**No.1 ゴールドの二重消費／消失**
- 該当: `src/lib/sync.ts` の `mergeLWW`（200〜213行）、`src/app/api/sync/push/route.ts` の user_status MERGE（65〜78行）
- 原因: `userStatus`（ゴールド・アイテム・ボックス等を含む1かたまり）を、新しい方で**まるごと上書き**している。ゴールドのような数値を「足し引き」でマージしないため、2端末で別々に使うと、後から同期した方の値で一方の消費が消える。
- 修正方針: ゴールドは通帳（`goldLedger`）から残高を再計算する、またはサーバー側で「加算/減算」を原子的に処理する、もしくはフィールド単位でマージする。
- 対応テスト: SYNC-005

**No.2 ボックス内の魚の編集が消える**
- 該当: `src/lib/types.ts` の `boxFish`、`src/lib/sync.ts`
- 原因: `userStatus.boxFish`（魚の配列）も No.1 と同じく、かたまり全体を後勝ちで上書きする。ボックス内の1匹だけの変更が、別端末の更新に飲み込まれる。
- 修正方針: ボックスの魚を独立した保存場所に分け、魚1匹（fishId）単位でマージする。
- 対応テスト: SYNC-006

**No.3 同期中のユーザー操作が上書きされる**
- 該当: `src/components/GameProvider.tsx`（起動時の同期処理 / `syncNow`）、`src/lib/sync.ts`（`clearFishList`→`putFishList`）
- 原因: 同期は「水槽をいったん空にして書き直す」方式。その最中にユーザーが操作すると、書き直しに上書きされて操作が消えうる。
- 修正方針: 同期中は「同期中…」表示を出し、その間の操作を一時的に止める（UIロック）。
- 対応テスト: SYNC-007

### 🟠 中

**No.4 削除した魚/単語が一時的に復活する**
- 該当: `src/lib/sync.ts`（69〜73行ほか）
- 原因: 削除済みを除外する仕組み（tombstone）はあるが、push と pull の順番が前後すると一瞬復活しうる。さらに古いデータの魚履歴には `fishId` が無く、除外から漏れる。
- 修正方針: 削除情報の優先度を上げる／旧データに `fishId` を補完する。
- 対応テスト: SYNC-003, SYNC-004

**No.5 clear+rewrite のすれ違い**
- 該当: `src/app/api/sync/push/route.ts`（54〜59行）
- 原因: push 時にクラウドのテーブルを `DELETE → INSERT` で書き直すため、別端末の pull とタイミングが重なると不整合が起きうる。
- 修正方針: 同一ユーザーの同時同期を直列化する／トランザクションで囲う。

**No.6 カスタム魚の削除が無確認で水槽の魚も消す**
- 該当: `src/components/GameProvider.tsx`（796〜806行付近の `removeCustomFish`）
- 原因: カスタム魚の定義を削除すると、水槽にいる同じ種類の魚も確認なしで全部消える。
- 修正方針: 削除前に「水槽の○匹も一緒に消えます」と確認ダイアログを出す。
- 対応テスト: MISC-010

**No.7 単語帳：絞り込み後の「全選択」に隠れた単語が混ざる**
- 該当: `src/components/WordManager.tsx`（330行付近）
- 原因: フィルターを変えても選択状態（selWords）に前の単語IDが残り、「全選択」時に表示外の単語まで対象になりうる。
- 修正方針: フィルター変更時に選択状態をリセット、または表示中の単語のみを対象に限定する。
- 対応テスト: WORD-014

**No.8 相棒バフが効いていない可能性（要確認）**
- 該当: `src/components/GameProvider.tsx`（264〜303行の `calculateOfflineEffects` 呼び出し）
- 原因: 放置ペナルティ計算に渡すバフが常に空（`{}`）。病気耐性・減衰軽減などが働いていない可能性。
- 修正方針: 実装意図を確認し、バフを集計して渡す or 仕様として削除を明記。
